# Test-Driven Development

## Required Workflow

1. Write tests FIRST
2. Run tests → Verify failure (Red)
3. Implement → Pass tests (Green)
4. Refactor if needed

## Commands

- make test-frontend
- make test-backend-py
- make test

## Prohibited

- Writing implementation before tests
- Modifying tests to make them pass
